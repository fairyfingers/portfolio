Tests

Prenez n’importe quelle application, n’importe quel produit, développez-y des fonctionnalités sur un ou deux ans, et constatez à quel point la moindre modification de code y devient si facilement critique. Il ne suffit que d’une simple ligne pour compromettre la sécurité de toute une application, ou même son fonctionnement.

Fort heureusement, il existe un moyen de s’assurer qu’aucune régression n’est induite par l’ajout ou la modification de code dans un projet. Ce moyen, ce sont les tests. Ils peuvent être de différente nature, tester un élément singulier ou un ensemble de fonctionnalités supposées fonctionner dans tel ou tel contexte.

C’est extrêmement pratique dans le cas où plusieurs développeurs travaillent sur la même application, ou lorsque le développement s’étend sur plusieurs mois ou années. J’ai moi-même eu l’opportunité de développer des tests lors de mon projet en cinquième semestre, Fork. Ces tests s’assuraient que le serveur de mon application pouvait toujours interagir avec la base de données à laquelle il était censé se connecter. Cela me permettait lors de chaque ajout de fonctionnalité, de m’assurer que les interactions entre mon serveur et ma base de données étaient toujours fonctionnelles. Et lorsque ce n’était plus le cas, c’est que le code que j’avais ajouté ou modifié n’était pas bon. À ce moment, je ne partageai pas mon code aux autres développeurs avant de m’être assuré que tous mes tests passaient de nouveau.

J’ai rencontré quelques développeurs qui considéraient les tests comme une perte de temps. S’il s’agit effectivement d’un effort en soi (puisqu’il faut les développer), le temps passé à les développer sera très largement rentable lorsque les premières régressions de ceux qui n’ont pas développé de tests apparaîtront dans leurs applications.

Je ne suis pas un testeur « fou », je teste ce qu’il est judicieux et important de tester, comme des fonctionnalités critiques dont l’utilisateur a besoin lors de ses nombreuses interactions avec mon produit par exemple. Je pense qu’il existe un juste milieu entre ne rien tester, et tout tester. Un milieu efficace et surtout maintenable.
